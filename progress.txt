# Codebase Patterns
- Submodule build.gradle pattern: plugins (java-library, io.freefair.lombok, jacoco, com.diffplug.spotless), spotless with googleJavaFormat 1.32.0 and OpenSearch license header
- Calcite version: 1.41.0, always exclude json-smart from calcite-core
- doctest:bootstrap fails in this env — use `-x :doctest:bootstrap -x :doctest:build` for full builds
- Do NOT add stub/test modules to publishedModules in root build.gradle
- Java 21 source/target set globally in allprojects — no need to redeclare in submodules
- Analytics SPI package structure: org.opensearch.analytics (EngineContext), .backend (EngineBridge), .exec (QueryPlanExecutor), .schema (SchemaProvider), .spi (AnalyticsBackEndPlugin)
- SPI interfaces use Object params to avoid server dependency — keep this pattern for stub code
- Parquet routing utility: ParquetIndexRouting in common/.../routing/ — extractIndexName(query) for PPL, extractIndexNameFromSql(query) for SQL, isParquetIndex(name). Shared across legacy/ and plugin/ modules.
- Cross-module sharing: put shared utilities in common/ module (both legacy/ and plugin/ depend on it)
- Cross-module function injection: legacy/ cannot depend on plugin/ — use Function<String,String> (sync) or BiConsumer<String,ActionListener<String>> (async) injected from SQLPlugin.java to bridge the gap
- Transport action registration: ActionType(NAME, Response::new) + HandledTransportAction with @Inject constructor, registered in SQLPlugin.getActions() as ActionHandler pair
- Calcite SQL Planner API: Frameworks.getPlanner(config) → parse(sql) → validate(sqlNode) → rel(validatedNode) → RelRoot.rel
- SQL parser config: Use Lex.MYSQL for case-insensitive identifiers and backtick quoting (timestamp is a reserved keyword)
- `match` is a reserved keyword in Calcite MySQL lex mode — cannot be registered as a parseable SQL function. Handle via SqlParseException catch with clear error message. Other search functions (match_phrase, etc.) are NOT reserved and parse fine through operator table registration.
- Build exclusions for local dev: `-x integTest -x :integ-test:yamlRestTest -x :doctest:bootstrap -x :doctest:build`
- Function validation pattern: UnsupportedFunctionValidator (RelVisitor + RexShuttle) checks SqlKind.OTHER_FUNCTION calls against EngineCapabilities. Validate BEFORE BoundaryScan replacement.
- PPL V3 datetime UDTs (EXPR_TIMESTAMP) are VARCHAR-backed. Standard Calcite TIMESTAMP is DATETIME family. PPLTypeChecker bridges via convertRelDataTypeToSqlTypeName(). DatePartFunction works with both at planning level. SpanFunction has runtime-only issue (instanceof ExprSqlType check).
- REST handler routing pattern: insert check in prepareRequest() BEFORE the existing `return channel ->` lambda. Return a different RestChannelConsumer for the new code path.
- Formatter-dependent code goes in protocol/ module (NOT common/) — common/ cannot depend on core/ or protocol/ without circular deps
- Parquet stub response: ParquetStubResponse in protocol/.../response/ — buildStubQueryResult(), formatAsSimpleJson(), formatAsJdbc(). Used by both PPL and SQL handlers.
- PPL uses SimpleJsonResponseFormatter (no alias, no status), SQL uses JdbcResponseFormatter (alias + status=200)
- sendResponse signatures: PPL handler (channel, status, contentType, content), SQL handler (channel, message, status)
- JDBC formatter uses legacyTypeName(): STRING→keyword, STRUCT→object, ARRAY→nested. All others use name(). SimpleJson uses typeName() which returns the ExprCoreType name directly (e.g. STRING→string).
- Absorption rules pattern: BoundaryScan (custom TableScan) + SimpleAbsorptionConfig (RelRule.Config without Immutables) + rules match LogicalXxx → BoundaryScan, check EngineCapabilities, call scan.absorb(). Rules registered via BoundaryScan.register().
- analytics-engine-stub/ has NO Immutables library — use SimpleAbsorptionConfig for RelRule.Config, not @Value.Immutable
