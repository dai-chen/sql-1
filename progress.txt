## Codebase Patterns

### PPL AST Structure
- PPL parser always wraps plans in Project(AllFields) â€” treat as passthrough
- PPL count() AST uses Literal(1) not AllFields â€” check for both
- Pipe commands chain as nested AST: Sort(Aggregation(Filter(Relation))) â€” process inside-out
- Join aliases (left=l right=r) are in the AST and must appear in SQL output
- SpanBin has field, value, unit; uppercase M = MONTH, lowercase m = MINUTE

### SqlNode Converter Patterns
- Each PPL pipe command wraps the previous result as a subquery (inside-out construction)
- Use SqlNodeDSL static factory methods: select(), from(), where(), groupBy(), orderBy(), etc.
- Use SqlBasicFunction.create() for unquoted function names â€” new SqlFunction() quotes them
- Deferred ORDER BY: use pendingOrderBy/pendingFetch fields to carry ORDER BY across pipe stages
- CAST to INTEGER for datetime EXTRACT functions â€” Calcite EXTRACT returns BIGINT
- PPLTypeCoercion handles implicit type coercion during validation â€” converter doesn't emit CASTs
- DynamicPPLToSqlNodeConverter uses SchemaPlus only for column enumeration (fields -, rename wildcards, fillnull all-fields)

### Function Mapping
- PPL functions map to SQL via FUNC_MAP (static map in converter)
- PPL-specific aggregates: firstâ†’PPL_FIRST, lastâ†’PPL_LAST, takeâ†’TAKE, percentile_approxâ†’UDF
- Arithmetic: addâ†’+, subtractâ†’-, multiplyâ†’*, divideâ†’/, modulusâ†’MOD
- Now-like functions: generate UTC string literals at transpile time (constant within query execution)
- replace()â†’REGEXP_REPLACE (PPL replace is regex-aware), regexp_matchâ†’REGEXP_CONTAINS
- Back-reference conversion in replace: \\Nâ†’$N (PPL Java-style â†’ Calcite style)
- SQL datetime keywords (CURRENT_TIMESTAMP, CURRENT_DATE, etc.) must NOT have parentheses

### Aggregation Patterns
- DATE_TRUNC argument order: DATE_TRUNC(field, UNIT) â€” Calcite BigQuery dialect, NOT PostgreSQL
- percentile_approx: UDF call, not PERCENTILE_CONT WITHIN GROUP
- bucket_nullable=false: add WHERE clause filtering null group-by keys
- TAKE: TAKE(field, N) UDF with count argument
- COUNT(*) not COUNT() â€” Calcite rejects zero-argument COUNT

### Subquery Wrapping Rules
- Eval aliases need subquery wrapping so downstream pipes can reference them
- When pipe stages conflict (WHERE after GROUP BY), auto-wrap as subquery
- ORDER BY lost in subquery wrapping â€” save/restore orderBy+limit around wrapAsSubquery
- head-then-sort: if limit already set from prior head, wrap as subquery before applying sort

### SELECT * EXCEPT/REPLACE
- fields- exclusion: SELECT * EXCEPT(col1, col2)
- eval column overriding: SELECT * REPLACE(expr AS col)
- bin field replacement: SELECT * REPLACE(bin_expr AS field) when alias matches field
- fillnull all-fields: SELECT * REPLACE(COALESCE(col, val) AS col, ...)
- rename: subquery wrap with alias mapping

### Test Harness
- V4 mode throws exceptions client-side (SyntaxCheckException, IllegalArgumentException) not server-side (ResponseException) â€” error-handling tests systematically fail
- Escaped double-quotes: V4 path needs query.replace("\"", "\"") preprocessing
- ORDER BY column stripping: Calcite adds ORDER BY columns to response even when not in SELECT
- JSON escaping: multi-line SQL from toSqlString() breaks JSON response serialization
- CalcitePPLPluginIT is flaky â€” ignore Â±1 fluctuations

### Engine Capabilities (do NOT modify engine)
- SqlLibrary.MYSQL/BIG_QUERY/SPARK/POSTGRESQL all registered â€” hundreds of functions available
- OpenSearch date/time fields â†’ standard SQL TIMESTAMP/DATE/TIME (no UDT)
- Metadata fields excluded from schema â€” SELECT * returns only user data columns
- SELECT * EXCEPT/REPLACE supported at SqlNode layer
- Custom PPL UDFs: pplFirst, pplLast, matchPhrase, percentile_approx, take, listAgg, valuesAgg

## Human Intervention â€” PPLToSqlNodeConverter Refactor (2026-03-10)

The string-based PPLToSqlTranspiler (58.5%, 1324/2264) was replaced with a SqlNode-based approach:

### What was created
- **PPLToSqlNodeConverter** â€” extends `AbstractNodeVisitor<SqlNode, Void>`, base converter for static commands (where, sort, head, eval, stats, dedup, join, lookup, subquery, append, top, rare, trendline, streamstats)
- **DynamicPPLToSqlNodeConverter** â€” subclass with SchemaPlus dependency for schema-dependent commands (fields include/exclude, eval column override, rename wildcard, fillnull all-fields, replace, bin, appendcol, graphlookup TVF)
- **SqlNodeDSL** â€” thin utility layer of static factory methods wrapping Calcite SqlNode constructors for declarative query building
- **PPLTypeCoercion** â€” extends TypeCoercionImpl with MYSQL_5 conformance, adds DATE+TIMEâ†’TIMESTAMP merge rule. Fires during Calcite validation to inject explicit CASTs.
- Wired into PPLIntegTestCase V4 routing path

### Key design decisions
- Converter produces SqlNode (not RelNode) â€” feeds into Calcite's validate â†’ rel pipeline
- Lighter dependency than V3's CalcitePlanContext (no RelBuilder, no RexBuilder)
- Converter does NOT track types or emit CASTs â€” PPLTypeCoercion handles coercion during validation
- Two-tier design: base (no schema) vs dynamic (schema-aware) for testability

### What was carried over from string transpiler
- Deferred ORDER BY pattern (pendingOrderBy/pendingFetch)
- Datetime function handling (ensureTimestamp, now-like UTC literals)
- PPL-specific aggregate mappings (PPL_FIRST, PPL_LAST, TAKE, percentile_approx)
- Function quoting fix (SqlBasicFunction.create for unquoted names)
