# Implementation Plan: 2.1 PPLToSqlNodeConverter + 2.2 PPLTypeCoercion

> Goal: Refactor PPL-to-SQL translation from string-based transpiler to SqlNode-based converter,
> add PPL implicit type coercion via custom TypeCoercionFactory, then resume Ralph loop to push
> PPL-to-SqlNode PoC to the limit using pure SQL (no UDT/UDF/TVF registration).

---

## Phase 1: Foundation — SqlNode DSL + Base Converter Scaffold

### Task 1.1: Create SqlNode DSL utility class

Create `api/src/main/java/org/opensearch/sql/calcite/utils/SqlNodeDSL.java` — static factory methods wrapping Calcite SqlNode constructors.

Minimum methods needed:
```
select(SqlNodeList), from(SqlNode), where(SqlNode), groupBy(SqlNodeList),
orderBy(SqlNodeList), limit(SqlNode), offset(SqlNode) → SqlSelect

star(), identifier(String...), literal(Object), table(String),
subquery(SqlSelect, String alias), asList(SqlNode...) → SqlNodeList

gt, lt, gte, lte, eq, neq, and, or, not → SqlBasicCall (comparisons)
plus, minus, times, divide, mod → SqlBasicCall (arithmetic)
call(String funcName, SqlNode... args) → SqlBasicCall
agg(String funcName, SqlNode... args) → SqlBasicCall
cast(SqlNode, SqlDataTypeSpec) → SqlBasicCall
as(SqlNode expr, String alias) → SqlBasicCall
between(SqlNode, SqlNode, SqlNode) → SqlBasicCall
in(SqlNode, SqlNodeList) → SqlBasicCall
caseWhen(List<SqlNode> whens, List<SqlNode> thens, SqlNode else_) → SqlCase
window(SqlNode agg, SqlNodeList partitionBy, SqlNodeList orderBy) → SqlWindow
```

**Test:** Unit test asserting `SqlNodeDSL.select(star()).from(table("t")).build().toSqlString()` produces `SELECT *\nFROM t`.

### Task 1.2: Create base PPLToSqlNodeConverter

Create `api/src/main/java/org/opensearch/sql/api/PPLToSqlNodeConverter.java`:
- Extends `AbstractNodeVisitor<SqlNode, Void>`
- Entry point: `public SqlNode convert(Statement statement)`
- Flatten AST chain → iterate → dispatch via visitor pattern (eliminates the triplicated if/else-if chain)
- Pipe chaining: each `visitXxx` returns a `SqlSelect`; the next command wraps it as a subquery via `SqlNodeDSL.subquery()`

Port FUNC_MAP from string→string to string→SqlOperator (or keep as string→string and resolve via `SqlNodeDSL.call()`).

Start with **5 core commands** only:
- `visitRelation` → `SELECT * FROM table`
- `visitFilter` → wrap + `WHERE condition`
- `visitProject` → `SELECT col1, col2, ...` (include-only, no exclude)
- `visitSort` → `ORDER BY`
- `visitHead` → `LIMIT`

**Test:** Unit test: `source=t | where a > 1 | fields b, c | sort b | head 10` → assert SqlNode.toSqlString() matches expected SQL.

### Task 1.3: Port expression visitors

Create `PPLExprToSqlNodeConverter` (or inline in the same class) for expression-level AST nodes:
- `visitLiteral` → `SqlLiteral`
- `visitQualifiedName` / `visitField` → `SqlIdentifier`
- `visitAlias` → `SqlBasicCall(AS)`
- `visitCompare` → `SqlBasicCall(>, <, =, ...)`
- `visitAnd/Or/Not` → `SqlBasicCall(AND/OR/NOT)`
- `visitFunction` → `SqlBasicCall(funcName, args)` with FUNC_MAP lookup
- `visitAggregateFunction` → `SqlBasicCall(aggName, args)`
- `visitIn/Between/Case/Cast` → corresponding SqlNode types
- `visitAllFields` → `SqlIdentifier.star()`

Port the ~60 FUNC_MAP entries. For special-case functions (if→CASE, log→LN, division→zero-guard), implement as SqlNode tree construction instead of string manipulation.

**Test:** Unit test each expression type in isolation.

## Phase 2: Core Commands — Reach Feature Parity with PoC

### Task 2.1: Aggregation (stats command)

- `visitAggregation` → `SELECT agg_exprs, group_exprs FROM (...) GROUP BY group_exprs`
- Handle span expressions in GROUP BY
- Handle `bucket_nullable=false` → add `WHERE field IS NOT NULL`

### Task 2.2: Eval command

- `visitEval` → wrap as subquery, `SELECT *, expr AS alias FROM (...)`
- Handle forward-reference inlining within same eval batch (replace SqlIdentifier references with their computed SqlNode)
- Column override (same name): `SELECT * EXCEPT(col) REPLACE(expr AS col)` if schema available, else `SELECT *, expr AS col` and let validator handle it

### Task 2.3: Dedup command

- Standard dedup → `ROW_NUMBER() OVER (PARTITION BY fields ORDER BY ...) AS _rn` + wrap + `WHERE _rn <= N`
- Consecutive dedup → `LAG(field) OVER (ORDER BY ...)` + filter where current != previous
- `keepempty=true` → `OR field IS NULL`

### Task 2.4: Join + Lookup

- `visitJoin` → `SELECT ... FROM left_subquery JOIN right_subquery ON condition`
- Handle join types: INNER, LEFT, RIGHT, FULL, CROSS
- SEMI/ANTI → rewrite to `WHERE EXISTS/NOT EXISTS (SELECT 1 FROM ...)`
- `visitLookup` → LEFT JOIN with REPLACE/APPEND output strategies

### Task 2.5: Subquery support

- `visitInSubquery` → `field IN (SELECT ...)`
- `visitScalarSubquery` → `(SELECT ...)`
- `visitExistsSubquery` → `EXISTS (SELECT ...)`
- `visitRelationSubquery` → subquery in FROM clause

### Task 2.6: Window functions (eventstats/streamstats)

- `visitWindow` → `SELECT *, agg OVER (PARTITION BY ... ORDER BY ...) FROM (...)`
- `visitStreamWindow` → running window with frame spec
- `visitWindowFunction` (expression) → `SqlWindow` node

### Task 2.7: Remaining core commands

- `visitRareTopN` → aggregate + ROW_NUMBER window + filter
- `visitTrendline` → SMA/WMA via window functions
- `visitAppend` → `UNION ALL`
- `visitBin` → CASE WHEN or FLOOR-based bucketing
- `visitReplace` → REGEXP_REPLACE
- `visitRename` → `SELECT col AS new_name, ...` (non-wildcard only in base)
- `visitFillNull` → `SELECT COALESCE(col, value) AS col, ...`
- `visitParse/Rex/Regex` → REGEXP_EXTRACT / REGEXP_CONTAINS

**Milestone:** All commands currently handled by PPLToSqlTranspiler are ported. Run ppl_it_rate.sh to measure baseline.

## Phase 3: DynamicPPLToSqlNodeConverter — Schema-Dependent Commands

### Task 3.1: Create DynamicPPLToSqlNodeConverter subclass

Create `api/src/main/java/org/opensearch/sql/api/DynamicPPLToSqlNodeConverter.java`:
- Extends `PPLToSqlNodeConverter`
- Constructor takes `SchemaPlus schema`
- Helper: `resolveColumns(String tableName) → List<String>` using `schema.getTable(name).getRowType()`

### Task 3.2: Override schema-dependent commands

- `visitProject` override: handle `fields -` (exclude) by enumerating all columns from schema, removing excluded ones
- `visitRename` override: handle wildcard rename by resolving matching columns from schema
- `visitFillNull` override: handle all-fields mode by enumerating columns from schema
- `visitEval` override: handle column override by enumerating other columns for explicit SELECT list
- `visitAddTotals/AddColTotals` override: identify numeric fields from schema for SUM

### Task 3.3: Wire into PplV4PlanningStrategy

Update `UnifiedQueryPlanner` / `PlanningStrategy` to use `DynamicPPLToSqlNodeConverter`:
```java
class PplV4PlanningStrategy implements PlanningStrategy {
    RelNode plan(String pplQuery) {
        Statement ast = parsePplToAst(pplQuery);
        SqlNode sqlNode = converter.convert(ast);  // DynamicPPLToSqlNodeConverter
        SqlNode validated = planner.validate(sqlNode);
        return planner.rel(validated).rel;
    }
}
```

## Phase 4: PPLTypeCoercion — Implicit Type Conversion

### Task 4.1: Create PPLTypeCoercion class

Create `core/src/main/java/org/opensearch/sql/calcite/type/PPLTypeCoercion.java`:
- Extends `TypeCoercionImpl`
- Constructor: `PPLTypeCoercion(RelDataTypeFactory typeFactory, SqlValidator validator)`

### Task 4.2: Override binaryComparisonCoercion

Add PPL-specific rules on top of Calcite defaults (which already handle STRING→NUMERIC comparisons):
- DATE + TIME → TIMESTAMP (Calcite only does DATE→TIMESTAMP, not DATE+TIME merge)
- Any other PPL-specific comparison coercion from `CoercionUtils.COMMON_COERCION_RULES`

### Task 4.3: Override builtinFunctionCoercion (if needed)

Evaluate whether Calcite's default `implicitCast()` (which already handles STRING→NUMERIC family, STRING→DATETIME family) is sufficient for PPL function signatures. If not, override to add PPL's distance-based closest-match logic.

### Task 4.4: Register via TypeCoercionFactory

In `UnifiedQueryContext.buildFrameworkConfig()`:
```java
.sqlValidatorConfig(SqlValidator.Config.DEFAULT
    .withTypeCoercionFactory((typeFactory, validator) ->
        new PPLTypeCoercion(typeFactory, validator)))
```

Use `SqlConformanceEnum.MYSQL_5` as base conformance to get BOOLEAN↔NUMBER coercion for free.

### Task 4.5: Test with cross-type comparison cases

Write unit tests for the specific cases that Calcite defaults don't handle:
- `WHERE date_field > time_field` → both cast to TIMESTAMP
- `WHERE boolean_field = '1'` → string literal to boolean
- Verify that standard cases (STRING vs NUMBER, DATE vs TIMESTAMP) work via Calcite defaults

## Phase 5: Integration + Regression Verification

### Task 5.1: Wire PPLToSqlNodeConverter into PPLIntegTestCase

Update V4 routing in `PPLIntegTestCase`:
- Instead of `PPLToSqlTranspiler.transpile(query)` → SQL string → send to endpoint
- Use `DynamicPPLToSqlNodeConverter.convert(ast)` → `SqlNode.toSqlString()` → send to endpoint
- Or: use full pipeline `convert → validate → rel → execute`

### Task 5.2: Regression comparison against baseline

Run `scripts/ppl_it_rate.sh` with the new SqlNode-based converter. Compare against Phase 0 baseline:
```bash
# Run with new converter
./scripts/ppl_it_rate.sh

# Diff against baseline
diff <(sort docs/dev/baseline-string-transpiler-failures.txt) \
     <(sort build/test-results/failures.txt)
```

**Success criteria:** Zero regressions — every test that passed with the string transpiler must also pass with the SqlNode converter. New passes (from PPLTypeCoercion fixing cross-type comparisons) are a bonus.

**This task is the Ralph loop itself.** Each iteration:
1. Run ppl_it_rate.sh
2. Diff against baseline to find regressions
3. Fix regressions in the converter
4. Repeat until zero regressions

Once zero regressions achieved, the old `PPLToSqlTranspiler` can be removed.
