# Sanity Test Agent Instructions

## Steps

1. **Check cluster health:**
   ```bash
   curl -s localhost:9200/_cluster/health
   ```
   If unreachable, tell the user to start it with `../start-cluster.sh` (and `../publish-sql-plugin.sh` if not done).

2. **Run the sanity test and generate report** from `analytics-engine-test/sanity/`:
   ```bash
   bash run-sanity-test.sh 2>/dev/null | python3 generate-sanity-report.py sanity-queries.json SANITY-REPORT.md
   ```

3. **Show the generated report:**
   ```bash
   cat SANITY-REPORT.md
   ```

## How It Works

- `sanity-queries.json` — 61 test queries with expected results (`expected.rows` + `expected.ordered`)
- `run-sanity-test.sh` — seeds 3 indices, runs queries, verifies results against expected values, outputs JSON
- `generate-sanity-report.py` — converts JSON output to markdown report

### Result Verification

Each query definition includes an `expected` field:
```json
{
  "category": "SELECT",
  "name": "select_where",
  "query": "SELECT * FROM sanity_test WHERE age > 30",
  "expected": {"ordered": false, "rows": [[32,"Seattle",1,"Alice"],[35,"Portland",2,"Dave"]]}
}
```

- `ordered: true` — exact row order match required (for ORDER BY queries)
- `ordered: false` — rows are sorted before comparison (for unordered results)
- `expected: null` — only checks HTTP 200 + no error (for NOW(), EXPLAIN, etc.)

### Categories (15)

SELECT, WHERE, ORDER BY, LIMIT / OFFSET, GROUP BY, HAVING, CASE, LIKE / REGEXP, Set Operations, JOIN, Subqueries, EXPLAIN, Full-text search, Functions: Math, Functions: DateTime, Window Functions

### Known Failures

| Category | Root Cause |
|----------|------------|
| Set Operations (UNION/UNION ALL) | SQL parser does not support UNION syntax |
| Full-text search + subquery_where | TaskResourceTrackingService thread reuse bug in filter delegation |
