fix(calcite): replace 5 temporary patches with clean minimal fixes

Replaces fragile temp patches (originally landed in 0f40eadfc and later
reverted) with production-quality fixes for the Calcite-based unified
SQL/PPL query path. Each fix is the smallest change that addresses the
root cause without changing observable behavior elsewhere.

1. FROM-less SELECT returning 0 rows (CalciteRelNodeVisitor.visitValues)
   Replace the empty-typed Values with Calcite's canonical
   LogicalValues.createOneRow(cluster). The downstream Project strips
   the dummy ZERO column. Fixes `SELECT 1`, `SELECT mod(3, 2)`, etc.

2. Strip Calcite's internal NUL separator from response field names
   (AnalyticsExecutionEngine.buildSchema)
   Calcite may produce field names like 'ISNULL(lastname)\0name' when
   disambiguating projected fields. Strip everything up to and including
   the NUL so users see only the alias.

3. DatetimeUdtNormalizeRule no longer crashes on aggregates over
   datetime UDT columns. Special-case Aggregate before super.visit():
   normalize children, then rebuild AggregateCall entries with null
   type so Calcite re-infers from the new input row type. Avoids the
   `aggCall type != inferred type` assertion in Aggregate.<init>.
   Extracts a toStdType helper to remove duplication.

4. HAVING / aggregate expression resolution
   (CalciteRexNodeVisitor.visitAggregateFunction)
   Replaces the previous fragile string-matching (with a dangerous
   prefix fallback) with a structural registry: CalcitePlanContext
   gains a Map<AggregateFunction, Integer> populated by
   CalciteRelNodeVisitor.visitAggregation after schema reordering.
   The rex visitor first does an O(1) registry lookup keyed on the
   AggregateFunction's structural identity (via @EqualsAndHashCode,
   which includes DISTINCT and condition), then falls back to a
   case-insensitive exact name match (no prefix matching), and throws
   IllegalStateException with the available field list if neither
   matches. Fixes `SELECT abs(MAX(age)) FROM t`,
   `SELECT MAX(age) + MIN(age) FROM t`,
   `... HAVING COUNT(*) > 1`.

5. NullOrder NPE in window functions (CalciteRexNodeVisitor)
   Window functions with ORDER BY but no explicit NULLS FIRST/LAST
   produced a SortOption with nullOrder = null, which NPE'd at the
   switch. Add a defensive null check that defaults to nullsFirst,
   matching the existing top-level ORDER BY default semantics in
   CalciteRelNodeVisitor.visitSort. Fixes
   `SELECT ROW_NUMBER() OVER(ORDER BY age) FROM t`.

Tests added:
- CalciteRexNodeVisitorTest: registry hit, name fallback, throws on
  unresolvable.
- AnalyticsExecutionEngineTest: NUL stripped, clean names unaffected.
- DatetimeExtensionSqlTest: aggregate over datetime/CAST plan-level.
- UnifiedQueryPlannerSqlV2Test: FROM-less SELECT plan-level.

All :api / :core / :sql unit tests pass on this branch.
